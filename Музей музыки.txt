1. Введение
1.1. Цель проекта

Создать интерактивное веб‑приложение для изучения музыкальных жанров, позволяющее пользователям просматривать описания жанров, прослушивать характерные аудиопримеры и загружать собственные MP3‑файлы.

1.2. Назначение продукта

Приложение предназначено для:

любителей музыки, желающих познакомиться с различными жанрами;

начинающих музыкантов для изучения стилистических особенностей;

образовательных учреждений в учебных целях;

коллекционирования и систематизации музыкальных примеров.

2. Требования к системе
2.1. Функциональные требования

Главная страница (/):

отображение списка известных исполнителей;

показ примеров песен для каждого исполнителя;

навигационное меню для перехода к другим разделам.

Страница жанров (/genres):

список музыкальных жанров с кратким описанием каждого;

возможность перехода к аудиопримерам по жанру.

Прослушивание примеров (/listen_examples/<genre>):

отображение списка песен для выбранного жанра;

встроенный аудиоплеер для прослушивания треков;

информация об исполнителе и названии песни.

Загрузка песен (/upload):

форма загрузки MP3‑файлов;

поля для ввода названия песни и исполнителя;

валидация формата файла (только MP3);

сохранение файла в папку uploads/;

запись информации о песне в data.json;

уведомления об успешной загрузке или ошибках.

Системные уведомления:

флеш‑сообщения об успехах (зелёный цвет);

сообщения об ошибках (красный цвет);

предупреждения (жёлтый цвет).

Обработка ошибок:

страница ошибки 404 для несуществующих маршрутов.

2.2. Нефункциональные требования

Производительность: время отклика сервера — не более 2 с.

Надёжность: сохранение данных при перезапуске сервера.

Безопасность: валидация загружаемых файлов, защита от загрузки вредоносного ПО.

Удобство использования: интуитивный интерфейс, адаптивный дизайн.

Масштабируемость: возможность расширения функционала без кардинальной переработки кода.

Совместимость: работа в современных браузерах (Chrome, Firefox, Safari, Edge последних версий).

3. Технические требования
3.1. Технологический стек

Бэкенд: Python 3.8+.

Веб‑фреймворк: Flask.

Утилиты: Werkzeug (для работы с файлами).

Хранение данных: JSON‑файл (data.json).

Фронтенд: HTML5, CSS3, JavaScript (базовый).

Хранение файлов: локальная папка uploads/.

3.2. Структура проекта

music-museum/

4. Функциональные модули
Модуль	Описание	Входные данные	Выходные данные

Страница жанров	Список жанров с описаниями	Нет	HTML‑страница с жанрами

5. Интерфейсы и маршруты
Маршрут	Метод	Описание
/	GET	Главная страница с артистами
/genres	GET	Страница жанров с описаниями
/listen_examples/<genre>	GET	Прослушивание песен выбранного жанра
/upload	GET/POST	Страница загрузки песен (GET — показать форму, POST — обработать загрузку)
/uploads/<filename>	GET	Доступ к загруженным файлам
Любой несуществующий маршрут	—	Страница ошибки 404
